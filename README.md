# Diabetes Mortality in the US–Mexico Border Region
## County-Level Spatial Analysis, 2018–2023

**Author:** Kaustubh Wagh  
**Data coverage:** 2018–2023  
**Sources:** INEGI, CONAPO, CDC WONDER  

---

## Overview

This repository contains all code to reproduce the county- and municipality-level spatial analysis of diabetes mortality across the US–Mexico border region. The analysis covers **4 US border states** (California, Arizona, New Mexico, Texas) and **6 Mexican border states** (Baja California, Sonora, Chihuahua, Coahuila, Nuevo León, Tamaulipas).

Two mortality measures are computed and mapped:

- **Crude Mortality Rate (CMR)** — unadjusted deaths per 100,000
- **Age-Standardized Mortality Rate (ASMR)** — adjusted using the WHO World Standard Population (2000–2025)

Spatial analyses include **Local Indicators of Spatial Association (LISA)**, **Getis-Ord G\* hotspot detection**, and **Moran scatterplots**, all produced at the county/municipality level.

## Execution Order

> ⚠️ **File 1 must be rendered before File 2.** File 2 loads `.RData` and `.rds` objects that are produced as outputs of File 1.

| Step | File | What it does |
|------|------|--------------|
| 1 | `1__County_Level_DM_Mortality_2018-2023.qmd` | Ingests all raw data, computes CMR and ASMR, merges US and Mexico data, loads shapefiles, saves processed `.RData`/`.rds` files |
| 2 | `2_Diabetes_USA_Mexico_Border_States_Maps.qmd` | Loads processed data, runs LISA, Getis-Ord, and Moran analyses, produces all 8 core maps |
| Optional | `Diabetes_border_map_county_level_Extra.qmd` | Additional choropleth variants (crude rate + ASMR) |
| Optional | `Diabetes_Burden_USA_Mexico_Border_Map_extra.qmd` | State-level and GBD-based burden maps |

## Data Sources

| Dataset | Source | Access |
|---------|--------|--------|
| Mexico mortality microdata (DEFUN, 2018–2023) | INEGI | [datosabiertos.salud.gob.mx](https://www.datosabiertos.salud.gob.mx) |
| Mexico population projections by municipality | CONAPO | [gob.mx/conapo](https://www.gob.mx/conapo) |
| US Multiple Cause of Death (county-level) | CDC WONDER | [wonder.cdc.gov](https://wonder.cdc.gov) |
| Mexico municipality & state shapefiles | INEGI Marco Geoestadístico | [inegi.org.mx](https://www.inegi.org.mx/app/biblioteca/ficha.html?upc=889463807469) |
| US county & state shapefiles (TIGER/Line 2024) | US Census Bureau | [census.gov](https://www.census.gov/geographies/mapping-files/time-series/geo/tiger-line-file.html) |
| GBD 2021 diabetes data (supplementary only) | IHME | [ghdx.healthdata.org](https://ghdx.healthdata.org) |

**Disease codes:**  
- Mexico (INEGI): LISTA_MEX codes `20D` (Type 2 diabetes) and `43I` (diabetes with complications)  
- USA (CDC WONDER): ICD-10 codes E10–E14 (diabetes mellitus)

**Age standardization:** WHO World Standard Population 2000–2025

## Setup

### 1. Clone the repository

```bash
git clone https://github.com/<your-username>/<repo-name>.git
cd <repo-name>
```

### 2. Create the `here` anchor

Open R in the repo root and run:

```r
install.packages("here")
here::set_here()   # creates a .here file at the repo root
```

### 3. Add raw data files

Download the raw data files from the sources listed above and place them in `data/raw/` following the folder structure shown in the Repository Structure section. The `data/processed/` folder will be populated automatically when File 1 is rendered.

### 4. Render the files in order

```bash
quarto render qmd/1__County_Level_DM_Mortality_2018-2023.qmd
quarto render qmd/2_Diabetes_USA_Mexico_Border_States_Maps.qmd
```

Or open each file in RStudio and click **Render**.

## Notes

- The `data/raw/` folder is listed in `.gitignore` and is **not** included in this repository due to data size and licensing restrictions. See the Data Sources table above to download each file directly.
- The `data/processed/` folder is also excluded from version control — it is regenerated by rendering File 1.
- Natural Earth boundary data is downloaded automatically at runtime via `rnaturalearth::ne_download()` and does not need to be downloaded manually.
- All map outputs are saved as 300 dpi PNG files (14 × 10 inches) suitable for publication.
